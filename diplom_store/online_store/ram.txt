PROFILE

/api/profile/ - используется в методе getProfile для получения данных профиля пользователя. После успешного выполнения запроса данные профиля
сохраняются в соответствующие поля: fullName, avatar, phone и email. Если произойдет ошибка при получении профиля, будет выведено предупреждение
"Ошибка при получении профиля".

/api/profile - используется в методе changeProfile для сохранения изменений профиля пользователя. Он принимает объект с данными профиля, включая
полное имя (fullName), аватар (avatar), телефон (phone) и электронную почту (email). После успешного сохранения профиля будет выведено оповещение
"Успешно сохранено". Если произойдет ошибка при обновлении профиля, будет выведено предупреждение "Ошибка при обновлении профиля".

/api/profile/password - используется в методе changePassword для сохранения изменений пароля пользователя. Он принимает текущий пароль (passwordCurrent)
и новый пароль (password). После успешного сохранения пароля будет выведено оповещение "Успешно сохранено". Если произойдет ошибка при сохранении пароля,
будет выведено предупреждение "Ошибка при сохранении пароля".

/api/profile/avatar - используется в методе setAvatar для загрузки и обновления аватара пользователя. Он принимает файл изображения в качестве данных
и отправляет его на сервер. После успешной загрузки изображения будет обновлено поле avatar. Если произойдет ошибка при обновлении изображения,
будет выведено предупреждение "Ошибка при обновлении изображения".

CATALOG

api/categories - используется в методе getCategories для получения списка категорий.

/api/catalog - используется для получения списка товаров в каталоге.
Он принимает несколько параметров: page (номер страницы), category (идентификатор категории),
sort (идентификатор выбранной сортировки), sortType (тип сортировки), filter (фильтры),
tags (список выбранных тегов) и limit (лимит товаров на странице).


ORDERS

/api/orders/active - используется для получения информации о активном заказе.
Он возвращает данные в формате JSON с информацией о заказе, такие как идентификатор (orderId),
дата создания (createdAt), полное имя (fullName), номер телефона (phone), адрес электронной почты (email),
тип доставки (deliveryType), город (city), адрес (address), тип оплаты (paymentType), статус (status),
общая стоимость (totalCost), список продуктов (products) и возможную ошибку оплаты (paymentError).

/api/orders - используется для создания заказа и получения списка заказов.

/api/orders/{orderId} - используется в методе confirmOrder для подтверждения заказа. Он принимает идентификатор заказа и отправляет данные о заказе для подтверждения.


CART

/api/cart - используется в методе getCartItems для получения содержимого корзины.

api /basket - используется для получения, добавления, удаления содержимого корзины.


TAG

/api/tags - используется в методе getTags для получения списка тегов. Он принимает параметр category, который указывает категорию товаров, для которой нужно получить теги.


BANNERS

/api/banners - используется в методе getBanners для получения списка баннеров.


PAYMENT

/api/payment/ - используется в методе submitPayment для отправки данных оплаты. Он принимает параметры оплаты, такие как имя (name), номер карты (number),
год истечения срока действия карты (year), месяц истечения срока действия карты (month), код безопасности (code). После успешной оплаты будет выведено оповещение
"Успешная оплата". Если произойдет ошибка при оплате, будет выведено предупреждение "Ошибка при оплате".

PRODUCT

/api/products/{productId} - используется в методе getProduct для получения данных о товаре. Он принимает идентификатор товара в качестве
параметра ({productId}). После успешного выполнения запроса данные о товаре сохраняются в объекте product. Если произойдет ошибка при
получении товара, будет выведено предупреждение "Ошибка при получении товара".

/api/product/{productId}/review - используется в методе submitReview для отправки отзыва о товаре. Он принимает идентификатор товара
в качестве параметра ({productId}) и объект с данными отзыва, такими как автор (author), электронная почта (email), текст отзыва
(text) и оценка (rate). После успешной отправки отзыва будет выведено оповещение "Отзыв опубликован". Если произойдет ошибка при
публикации отзыва, будет выведено предупреждение "Ошибка при публикации отзыва".

/api/sales - используется в методе getSales для получения данных о распродажах. После успешного выполнения запроса данные о распродажах сохраняются в
поле salesCards. Если произойдет ошибка при получении данных о распродажах, ошибки не обрабатываются или выводится предупреждение.

/api/products/popular - используется в методе getPopularProducts для получения списка популярных товаров.

/api/products/limited - используется в методе getLimitedProducts для получения списка лимитированных товаров.
